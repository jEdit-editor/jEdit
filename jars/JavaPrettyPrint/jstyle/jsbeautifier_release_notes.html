JSBeautifier - Release notes

 
 
 
 

In Beautifier 1.1.1 :

  • Fixed an important (although up to now unnoticed...) bug in multi-line statements that come right after 'for', 'if', ... statements.

    • In Beautifier 1.1.0 :

    • Lines within multi-line statements are now indented RELATIVELY to each other.
    • Static arrays (i.e. a = {1, 2, 3}) are now detected and handled properly.
    • Multiple-line comments now retain their original look, which is important for commented code.
    • 'switch' blocks now automatically contain an additional indent, making them (IMHO) more readable.
    • Added ability to flush 'switch' blocks (ala the Java Code Convention) via the option '-fs'.
    • Added ability to add an extra indent to brackets, via option '-ib' (due to user interest).

    In Beautifier 1.0.6

    • MAJOR improvement in the indentation of inner classes.

    In Beautifier 1.0.5

    • Fixed stream buffer flushing bug that was introduced in 1.0.4 .
    • Fixed bug introduced in 1.0.4, regarding use of semicolons within parenthesies.

    In Beautifier 1.0.4

    • Beautifier now accepts specific file names, in addition to redirection from input-stream.
    • Fixed serious bug in anonymous inner classes
    • Other minor bug fixes.

    In Beautifier 1.0.3

    • Fixed indentation bug in 'case' statements.
    • Other minor bug fixes.

    In Beautifier 1.0.2

    • Fixed indentation bug in phrases containing headers after a '}'.
    • Fixed indentation bug that occured in multiple catch() phrases.

    In Beautifier 1.0.1

    • Beautifier nows makes it possible to customize comment indenting.
    • You can now decide to add additional indentation at the begining of every line within a /* ... */ comment.

      • The default indent is 1 space - making standard javadoc comments look much nicer.
