1.8 and beyond To do:
  • jEdit has been through 8 major revisions, and since version 1.0 it has quadrupled in size... so all the bloat and dead code needs to be removed, and the old code needs to be overhauled and made more compact:
    • Menus should be loaded from an XML file, with a TurboNotes like MenuBarModel and ToolBarModel
    • XML files should be used instead of properties (with properties, lots of stuff is stored twice, for example the recent file list is stored once in the properties, and also in the recent vector. With XML, this can be avoided)
    • Rewrite C, HTML, TeX token markers to use PerlTokenMarker-like structure
    • Ask Mike Dillon to do the same to TSQL if necessary
    • Drop support for Swing < 1.1 (removeAll() hack, maybe others)
    • Eventually, switch to JDK 1.2 and add drag and drop, remove deprecates, change RMI to CORBA, add im/bidi support, use collections, etc
  • Possible syntax package improvements:
    • Finish Perl mode (multiline s/// & co)
    • Token markers should be able to re-mark a specific region of the document, instead of being limited to entire lines
    • Tokens should be cached so that constant tokenizing isn't necessary
    • XMode integration
    • More speed is always needed
  • Possible text area improvements:
    • Folding
    • Rectangular selection
    • More speed is always needed
  • Possible core improvements:
    • Embeddability
    • Macro language
    • XML indent
    • Piece content
