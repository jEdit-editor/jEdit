How do I Change the Compiler Used by `Tools->Compile'?

 This is detailed in the Invoking Mode Specific Compilers help topic.

Why can't I change directories in the console?

 At the moment, Java doesn't support changing the current working directory (user.dir).

Why do certain commands not work in the console on Windows?

 On Windows 95 and 98, some commands, such as dir, del, mkdir, type and several others are internal to COMMAND.COM, the MS-DOS command interpreter. To invoke them you must enter the following in the console: 
command.com /k command
On Windows NT, the same commands are internal to CMD.EXE, the 32-bit command interpreter. To invoke them you must enter the following in the console: 
cmd /k command

Why do I get OutOfMemoryErrors?

 When you have lots of buffers open, there might not be enough free space in the Java heap, even though lots of RAM is available. The solution is to increase the Java heap memory allocation.

On Windows, enter the following in an MS-DOS Prompt before starting jEdit, or place it in your AUTOEXEC.BAT to have it done every time the system starts: 

set JEDIT=-mxnnm
On Unix systems with a Bourne shell (ash, bash, jsh, sh, zsh) enter the following in a command line before starting jEdit, or place it in your .login or .profile to have it done every time you log in: 
set JEDIT=-mxnnm; export JEDIT
On Unix systems with a C shell (csh, tcsh) enter the following in a command line before starting jEdit, or place it in your .csh.login or .csh.profile to have it done every time you log in: 
setenv JEDIT -mxnn
Where nn is the size of the Java heap, in megabytes. The default is 32 Mb - if you like to open lots of files at once, 64 might be better.

Why does jEdit hang when exiting?

There is a bug in some JDKs where calling ServerSocket.close() will hang the JVM. This is documented as bug 4107121 in the JDC bug parade (see http://developer.java.sun.com).

The only workaround is to disable the jEdit server in `Tools->Global Options->General'.

Why does the display mess up when editing large files?

Due to an AWT limitation or Swing bug (or maybe both) the Swing JTextComponent component has problems editing documents greater than about 2000 lines. The problem is that the viewscreen turns gray and GUI elements are repainted incorrectly. Until Sun fixes this, other editors will have to be used to edit very large files.

Why doesn't overwrite mode work?

 Overwrite mode doesn't appear to work on Windows JDK 1.2. I am working on a fix; it will be available as soon as possible.

If overwrite mode doesn't work on other systems, the most likely problem is that your jEdit is still loading the AltGrPatch plugin that was obsoleted as of version 1.6pre4. Delete any copies of altgr.jar from the system and user plugin directories (see the jEdit Plugins help topic for details).

Why doesn't the AltGr key work?

JDK 1.1.7 and earlier had a bug where the AltGr key generated the wrong keycode. Upgrade to JDK 1.1.8 or later to fix this problem.

Why doesn't the `Open in WWW Browser' command work?

 On Unix and Windows, make sure that either the full path to the browser is entered in `Tools->Global Options->General', or that the browser's directory is listed in your PATH.

On Unix, it is recommended that the `jedit_moz_remote' wrapper for Netscape Navigator be used. The script relies on the `netscape' executable being in the PATH.

On the MacOS, the `Open in WWW Browser' feature does not work at all. If anybody can confirm otherwise, I will be happy to post information here.

Why don't the Alt key mnemonics work?

On some JDK versions, the AWT incorrectly sets the modifiers of KEY_TYPED events to 0 even if the Alt key is pressed. The end result is that the keystroke is inserted in the document. This is an AWT bug, and as far as I know the only way to fix it is to upgrade to a later JDK version.

Why is jEdit so slow?

 Because jEdit is written in Java, it's performance will always be slower than native applications. A fast system with a good JVM will narrow the performance gap.

On all systems, a speedup can be achieved by installing the latest Swing version. Swing 1.1.1 seems to be much faster than either Swing 1.1beta3 or Swing 1.1final.

Installing more RAM may improve performance - more RAM will result in less garbage collection.

On Linux, try installing the TYA JIT <ftp://gonzales.cyberus.ca/pub/Linux/java>.
