Introduction

 In order to allow various components to communicate without having knowledge of each other's internals, jEdit has an event system, much like that of the Java AWT toolkit. The event system consists of various event sources which fire event objects at any registered event listeners.

Overview

 Each event source generally defines the following three methods for each event type it can fire:
  • addTypeListener(TypeListener listener) - adds a listener for events of type Type to that event source
  • removeTypeListener(TypeListener listener) - removes a listener for events of type Type from that event source
  • fireTypeEvent(TypeEvent evt) - fires the specified event to all registered listeners
An event is an object that carries various pieces of information, such as the event subtype, the time when the event was fired, and what other objects, if any, were involved. All jEdit events extend the org.gjt.sp.jedit.event.AbstractEditorEvent class.

A listener is an interface that defines one method for each event subtype. All jEdit listeners implement the java.util.EventListener interface.

An adapter is a special type of listener. It is a class that implements a listener interface, with all empty methods. It allows one to extend that class, only overriding the methods of interest.

To receive events, one would implement a listener (either by implementing the interface directly, or extending an adapter) and add that listener to the event source with an addTypeListener() method.

When receiving events is no longer necessary, the removeTypeListener() method can be used to remove a listener from the event source.

Editor Events

The most frequently used event type is the editor event. Editor events are fired by the jEdit class and are used to notify listeners of the following:
  • Buffers being created and closed
  • Views being created and closed
  • Properties being changed
The following methods relate to editor events:
  • jEdit.addEditorListener()
  • jEdit.removeEditorListener()
  • jEdit.fireEditorEvent() (not normally used)
The following classes and interfaces relate to editor events:
  • EditorEvent
  • EditorListener
  • EditorAdapter

Buffer Events

 Buffer events are fired by instances of the Buffer class and are used to notify listeners of the following:
  • The dirty flag being set and cleared (buffer being modified and saved)
  • Markers being added and removed
  • The edit mode being changed
The following methods relate to buffer events:
  • Buffer.addBufferListener()
  • Buffer.removeBufferListener()
  • Buffer.fireBufferEvent() (not normally used)
The following classes and interfaces relate to buffer events:
  • BufferEvent
  • BufferListener
  • BufferAdapter

View Events

 View events are fired by instances of the View class and are used to notify listeners of the following:
  • The view's buffer being changed
  • The view's current error being changed
The following methods relate to view events:
  • View.addViewListener()
  • View.removeViewListener()
  • View.fireViewEvent() (not normally used)
The following classes and interfaces relate to view events:
  • ViewEvent
  • ViewListener
  • ViewAdapter

The EventMulticaster Class

 The EventMulticaster class is the core of the jEdit event system. It manages lists of event listeners and fires events at them. If you wish to implement your own event source and/or event type, this class is the easiest way to do so. It implements the following methods of interest:
  • addListener(AbstractEventListener listener) - adds an event listener
  • removeListener(AbstractEventListener listener) - removes an event listener
  • fireEvent(AbstractEditorEvent evt) - fires the event to all listeners that can receive it
Multiple types of listeners can be registered with each multicaster. When an event is fired, listeners that can't understand that event type are skipped.
