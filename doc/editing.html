Introduction

 This file describes jEdit's most basic functions. It assumes you are familiar with another text editor, for example GEdit on Unix, Notepad on Windows or SimpleText on the MacOS.

Starting jEdit

jEdit can be started from the command line to edit one or more files like this:
jedit <file1> <file2>...
If the file name contains a hash (#) the remainder of the file name after the hash character is the name of a marker that will be jumped to. (See the Markers help topic for more information.

A number of `switches' can also be passed to jEdit.

-version: Display jEdit version and exit
-usage: Display a command line usage message and exit
-nousrprops: Don't load user properties
-noserver: Don't start server
-nodesktop: Ignore saved desktop
-nosplash: Don't show initial splash screen
-usrprops=<file>: Load user properties from <file>
-portfile=<file>: Save server port to <file>
-desktopfile=<file>: Save desktop to <file>
-readonly: Open files read only
--: End of options
Because the `-' character is reserved for switches, you must enter the `--' parameter prior to any file names that start with `-'.
jedit -- -filename
jEdit can also be started from a GUI or command line with no parameters, in which case it will either open any files that were open in the last session, or if the `save desktop' feature is disabled, a blank `Untitled' file.

Opening Files

Once jEdit is started, files can be opened in a number of ways:

  • By starting another copy of jEdit. - if jEdit is started when there is another copy running, the files are opened in the already running copy.
  • With the `File->Open File' command. - This displays a standard (or not so standard, depending on the look and feel selected in `Tools->Options->General') file selection dialog box.
  • With the `File->Open URL' command. - This displays a text entry dialog where a URL can be entered. The name `Open URL' is somewhat misleading, since file names, as well as URLs, can be entered. The exact protocols available depend on the Java implementation - most support ftp: and http:.
  • With the `File->Open Selection' command. - This opens the file or URL specified by the currently selected text.
  • With the `File->Open Recent' menu. - This menu displays the most recently opened files, 8 by default but this can be changed in `Tools->Options->General.
  • With the `File->New File' command. - This command creates an empty file named `Untitled-n' where n is the number of `Untitled' files open so far. The new file isn't actually created on disk until you save; see below.

Saving Files

 Any changes made to open files aren't actually stored to disk until you save. (But see below about the auto save feature). To save the file to disk, do one of the following:
  • Close the file or exit jEdit. - If you close a file or exit jEdit, you will be asked if any unsaved changes should be saved.
  • With the `File->Save' command. - This command saves the file to disk without further user input. If the file is `Untitled', you will be prompted to enter a file name first.
  • With the `File->Save As' command. - This command displays a file selection dialog box where an alternative path name to save the file to can be entered or selected.
  • With the `File->Save to URL' command. - This command displays a text entry dialog box where a URL to save the file to can be entered. As with `Open URL', file path names are accepted too. The available protocols and the exact behaviour of URL uploading depend on the Java implementation.

Autosave

If jEdit crashes, there is still a good chance that you will be able to recover any unsaved changes.

Every 15 seconds (the interval can be changed in `Tools->Options->General') a copy of every open file with unsaved changes is written out to #filename# where filename is the name of the file. If jEdit crashes (or you do a file listing in a directory which contains files open in jEdit) you will see these autosave files.

If jEdit has truly crashed, you can try opening these autosave files to see if they contain what you were editing before jEdit crashed. If so, you can overwrite the original copy of the file.

You might also see files named #filename#+tmp+# where filename is a file you were editing. A file named that means that jEdit crashed while writing out a new autosave - the old one should still be in #filename#. The +tmp+ version is probably useless, since it is incomplete.

Backups

Sometimes you start editing a file, save it several times, then decide that you don't like the changes.

The first time you save a file after opening it, jEdit copies the old vesion to filename~ where filename is the name of the file being saved. Subsequent saves don't update the backup - you must close the file and save it again for that to happen. The result is that the backup always contains the file, as it was before you started editing it.

Multiple backups (named filename~1~, filename~2~ ... filename~10~ and so on, 1 being the newest and 10 being the oldest) can enabled in `Tools->Options->General'.

Sending and Printing Files

Files can be sent via e-mail with the `File->Send' command. `File->Print' can be used to print the file to a printer (or on some operating systems, to a PostScript file).

Cut, Copy and Paste

 jEdit has several commands for moving and copying text. They are:
  • Edit->Cut - this command places the selection in the clipboard and removes it from the document.
  • Edit->Copy - this command places the selection in the clipboard and leaves it in the document.
  • Edit->Paste - places the clipboard contents in the document, replacing the selection, if any.
  • Edit->Paste Previous - displays a list of previously copied and pasted strings available for insertion. By default, the last 100 clippings are remembered - this can be changed in `Tools->Options->General'.

Undo and Redo

 In case you make a mistake, the `Edit->Undo' command undoes the last editing action. Up to 100 previous actions can be undone. If you undo too much, try `Edit->Redo'.

Keyboard Shortcuts

 Within jEdit, keyboard shortcuts are written as modifier+key, where modifier is S for Shift, C for Control, or A for Alt. key is either a letter, or a named key such as PageUp.

Some keyboard shortcuts consist of more than one keystroke, for example `Delete Paragraph' is C+d C+e. Both key strokes must be pressed in the respective order.

Menu Item Shortcuts

The most frequently used menu item commands have keyboard shortcuts. They are listed to the right of the menu item's label.

Menu Mnemonics

Pressing the Alt key in combination with one of the underlined characters in the menu bar activates the respective menu. Once a menu is open, an item in it can be selected by pressing the underlined character in its label.

While a menu is open, the arrow keys can be used to select menu items, and Space or Enter will activate the selected menu item.

The F10 key can be used to activate the menu bar for menu selection via arrow keys.

Dialog Shortcuts

All dialogs have an OK button which can be activated by pressing the Enter key.

Most dialogs also have a Cancel button which can be activated by pressing the Escape key.

Tab and Shift+Tab can be used to shift the keyboard focus to the next and previous component in a dialog box.

Text Editing Shortcuts

The following shortcuts are available in most text components, including the main editor component and the text fields in various dialogs.

If Shift is held down in combination with any of these keys, the selection is extended, rather than the caret being moved.
Backspace Deletes the character before the caret. If there is a selection, it will be deleted instead.
Delete Deletes the character after the caret. If there is a selection, it will be deleted instead.
Left, Right, Up, Down Moves the caret in the respective direction.
C+Left, C+Right Moves the caret one word backward and one word forward, respectively.
Home Moves the caret to the start of the line.
End Moves the caret to the end of the line.
C+Home Moves the caret to the start of the text.
C+End Moves the caret to the end of the text.
PageUp Moves the caret up one screenful of text.
PageDown Moves the caret down one screenful of text.

Using Multiple Editor Windows

By default, jEdit has only one editor window, or view, open. All open files, or buffers are opened in that view. However, new views can be created with the `File->New View' command, and they can be closed with `File->Close View'. The `Buffers' menu can be used to select which buffer the current view is to edit. Several views can be editing the same buffer, in which case changes in one view will propagate to all others editing the same buffer.

The jEdit Server

 When jEdit is first started, it creates a new server thread that listens on a free TCP/IP socket. Subsequent invocations of jEdit connect to that socket and inform it of any files that are to be opened. This technique cuts down on memory usage because there is only one instance of jEdit open at any one time.

The server can be disabled either on a `one-time' basis with the `-noserver' switch, or permanently in `Tools->Options->General'.

The Desktop File

When jEdit exits it saves the list of open files to the `desktop' file, so that the next time jEdit is started with no command line parameters, it can reopen those files.

The desktop can be disabled either on a `one-time' basis with the `-nodesktop' switch, or permanently in `Tools->Options->General'.
