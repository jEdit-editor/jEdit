What's New In jEdit 1.5

  • User Interface Improvements
  • Backup System Improvements
  • Buffer Options Dialog Box
  • New Plugin Loader and API
  • Modular Syntax Colorizing Package
  • Bug Fixes

User Interface Improvements
  • The `console' (component for invoking external commands) is no longer a separate window, instead each view has one built in. The `Tools->Show/Hide Console' command can be used to toggle it's visibility.
  • The history text field component has been changed to display it's choices as a combo box, instead of the non-intutive arrow key system previous versions used.
  • In previous versions, `Search->Find Selection' searched for the selected string immediately. Now, it displays a new search & replace dialog, with the selected string already entered.
  • URLs entered in the console are now opened in the editor (for example, try entering file:C:\autoexec.bat).
  • The options dialog box has a new color settings pane.
  • Menu items and the tool bar have a new look.

Backup System Improvements

 The backup filename prefix and suffix, and the directory where backups are saved can now be specified in the options.

Buffer Options Dialog Box

 A new `Tools->Buffer Options' command displays a dialog box for setting the tab size and soft tabs setting on a per-buffer basis. The edit mode and line separator can also be set in that dialog.

New Plugin Loader and API

 A new interface is provided that plugins must implement. It allows for greater flexibility, and provides hooks for future features such as enabling/disabling plugins on the fly and getting information about plugins at runtime.

The class loader has been completely overhauled. It is now much faster because it only loads classes as they're needed - previous versions loaded all classes in one go.

Modular Syntax Colorizing Package

The syntax colorizing code is now completely modular and independent of jEdit, which means that other Java applications can use it to get syntax colorizing capabilities.

Bug Fixes

 jEdit 1.5 fixes the following bugs present in 1.4:
  • HistoryTextField had several problems, using the history to make a selection would often leave the text field in a weird state
  • `File->Line Separator' menu wasn't updated for all views editing a buffer
  • The current file wasn't restored properly by the session management code
  • Print command would sometimes print stuff other than the file name in page headers
  • `.cmd' files are no longer edited in batch file mode because they have a different syntax
  • Batch file token marker colorized stuff like C:\bob\fred.exe incorrectly
  • Invalid file names in compiler errors caused problems
  • Reload command didn't clear the dirty flag and could throw an array index out of bounds exception
  • Non-existent files are no longer opened twice by the console
