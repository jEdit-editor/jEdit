Starting jEdit

 jEdit can be started from the command line to edit one or more files like this:
jedit <file1> <file2>...
If the file name contains a hash (#) the remainder of the file name after the hash character is the name of a marker that will be jumped to in the opened file.

A number of `switches' can also be passed to jEdit.

-version: Display jEdit version and exit
-usage: Display a command line usage message and exit
-nousrprops: Don't load user properties
-noserver: Don't start server
-nodesktop: Ignore saved desktop
-nosplash: Don't show initial splash screen
-usrprops=<file>: Load user properties from <file>
-portfile=<file>: Save server port to <file>
-desktopfile=<file>: Save desktop to <file>
-readonly: Open files read only
--: End of options
Because the `-' character is reserved for switches, you must enter the `--' parameter prior to any files whose names start with `-'.
jedit -- -filename
jEdit can also be started from a GUI or command line with no parameters, in which case it will either open any files that were open in the last session, or if the `save desktop' feature is disabled, a blank `Untitled' file.

The jEdit Server

When jEdit is first started, it creates a new server thread that listens on a free TCP/IP socket. Subsequent invocations of jEdit connect to that socket and inform it of any files that are to be opened, and the thread opens those files. This technique cuts down on memory usage because there is only one instance of jEdit open at any one time.

The server can be disabled either on a `one-time' basis with the `-noserver' switch, or permanently in the Options dialog box.

The Desktop File

When jEdit exits it saves the list of open files to the `desktop' file, so that the next time jEdit is started, it can reopen those files.

The desktop can be disabled either on a `one-time' basis with the `-nodesktop' switch, or permanently in the Options dialog box.

Buffers

A buffer is an in-memory copy of an open file. All editing operations are performed on the buffer - the original remains unchanged on disk until the buffer is `saved'. The number of open buffers is only limited by available memory.

Buffer-Local Properties

Several paremeters such as the tab size can be set on a per-buffer basis with buffer-local properties.When a file is loaded, jEdit looks for something like the following in the first 10 lines:
(:name=value:...)
Any number of name/value pairs can be specified.

The following properties are supported:
closeBrackets A list of closing brackets, used by the Locate Bracket and Select Code Block commands. Default is `)]}'.
compiler The compiler to use for the buffer. Default is mode specific.
mode The edit mode to use for the buffer. Default is none.
make The name of the `make' tool. The global default can also be specified in the Options dialog box.
maxLineLen The maximum line length, used by the Format command. Default is 72.
noTabs In edit modes with automatic indentation (all the edit modes included with jEdit except for Text) if set to `yes', Tab characters will not b eused, and an equivalent number of spaces will be inserted instead. The Shift Indent Left and Shift Indent Right commands use this property as well.
noWordSep A list of non-alphanumeric characters that are not to be treated like word separators by the Expand Abbreviation command. Default is none, however the C, C++ and Java edit modes define this to `._' by default.
openBrackets A list of open brackets, used by the Locate Bracket and Select Code Block commands. Default is `([{'.
tabSize The tab width, in character units. Default is 8.

Views

A view is an editor window, the user interface for editing buffers. Each view can edit one buffer at a time, and any number of views can be editing a buffer. A change in one view is automatically propagated to all others editing the same buffer.

The user interface of views should be pretty self explanatory - there is only one part that needs explaining, the status bar. The status bar is the strip of writing at the bottom. A sample status bar looks like this: 

(--:**) editing.html (HTML, Line 32/53 60%)
The elements, from left to right, are:
  • (--) New file flag. If the current file is not on disk, this is **, -- otherwise.
  • (:) Read only flag. If the current file is read only, this is /, : otherwise.
  • (**) Changed flag. If the current file has been changed since it was last saved, this is **, -- otherwise.
  • (HTML) The current file's edit mode. |XXX edit modes|
  • (Line 32/53 60%) The current line number and the total number of lines in the file. The percentage is the ratio of the two.
The New View command can be used to create a new view, and the Close View command closes the current view. The `Buffers' menu can be used to select which buffer the current view is editing.

Edit Modes

 To implement indentation and syntax colorizing, jEdit has edit modes - each edit mode defining behaviour for editing some type of file. Namely, edit modes can define the following behaviour:
  • Indentation - what to do when the Tab key is pressed
  • Syntax Colorizing - jEdit has the capability to display different lexical tokens in source code using distinctive colors. Each file type is colorized accordingly
  • Buffer-Local Properties - defaults for buffer-local properties, for example the compiler to use
When a file is opened, jEdit looks at it's extension, file name and first line of text to determine which edit mode to use. The following table lists all edit modes included with jEdit, their internal names (which is the value that can be used in the `mode' buffer-local property) and how they are selected.
NameInternal Name Description
AMSTeX amstex For editing AMSTeX files. This mode is automatically selected for files whose names end with `.tex' and which contain `\documentstyle' in the first 100 lines, but which do not contain `\begin'. Default compiler is `amstex'.
Auto Indent autoindent Implements semi-intelligent indentation. All other modes subclass this.
Batch File bat For editing MS-DOS Batch Files. This mode is automatically selected for files whose names end with `.bat'.
C c For editing C source. This mode is automatically selected for files whose names end with `.c' or `.h'. Default compiler is `cc'.
C++ cc For editing C++ source. This mode is automatically selected for files whose names end with `.cc', `.cpp', `.cxx' or `.C'. Default compiler is `c++'.
HTML html For editing HTML files. This mode is automatically selected for files whose names end with `.html', `.htm' or `.shtml'.
Java java_mode For editing Java source. This mode is automatically selected for files whose names end with `.java'. Default compiler is `javac'.
JavaScript javascript For editing JavaScript source. This mode is automatically selected for files whose names end with `.js'. Also, HTML mode uses this mode to colorize scripts inside <script> tags.
LaTeX latex For editing LaTeX files. This mode is automatically selected for files whose names end with `.tex' and that contain `\begin' in the first 100 lines. Default compiler is `latex'.
Makefile make For editing Makefiles. This mode is automatically selected for files named `Makefile'.
Shell Script sh For editing UNIX shell scripts. This mode is automatically selected for files whose first line is `#!' followed by various shell paths.
TeX tex For editing plain TeX files. This mode is automatically selected for files whose names end with `.tex' but which do not contain either `\begin' or `\documentstyle' in the first 100 lines. Default compiler is `tex'.
